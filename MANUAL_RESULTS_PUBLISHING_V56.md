# TM6 Bonus v56 — ручне оновлення результатів

Результати в застосунку більше не читаються напряму з живих формул Google Таблиці.
Застосунок показує останню опубліковану копію даних і змінює її лише після ручного запуску `updateResultsSnapshot`.

## Що копіюється під час публікації

- `Goals`
- `Аркуш2`
- `Transformation` (якщо існує)
- `CreditMetrics` (якщо існує)

Apps Script створює приховані аркуші з префіксом `_TM6_PUBLISHED_`. Не редагуйте їх вручну.

## Налаштування

1. Замініть код Apps Script файлом `integrations/google-sheets/Code.gs`.
2. Збережіть проєкт Apps Script.
3. Повторно опублікуйте Web App:
   `Deploy → Manage deployments → Edit → New version → Deploy`.
4. Перезавантажте Google Таблицю. У верхньому меню з’явиться:
   `TM6 Bonus → Оновити результати`.
5. Натисніть цю команду один раз і надайте Apps Script дозволи.

## Як зробити видиму кнопку на аркуші

1. У Google Таблиці виберіть `Вставити → Малюнок`.
2. Намалюйте кнопку з текстом `Оновити результати`.
3. Натисніть три крапки на малюнку → `Призначити сценарій`.
4. Введіть назву функції без дужок:
   `updateResultsSnapshot`

Після натискання кнопки Google Таблиця покаже повідомлення з часом публікації.

## Нова логіка

- Формули й вихідні аркуші можуть змінюватися будь-коли.
- Користувачі продовжують бачити попередній опублікований результат.
- Нові значення з’являються в застосунку лише після `Оновити результати`.
- Автоматичне опитування раз на 60 секунд, оновлення при фокусі та поверненні у вкладку прибрані.
- Уже відкрита сторінка застосунку побачить нову публікацію після повторного відкриття сторінки або перезавантаження застосунку.
